# Milestone 2 – Project Specification

## Detection Signals

### Signal 1 – Groq LLM Classification

**Purpose:** Uses the Groq LLM to analyze writing style, tone, coherence, and language patterns to estimate whether the text appears AI-generated.

**Output:** A confidence score from **0.0 to 1.0**, where:

- 0.0 = Strong evidence of human writing
- 0.5 = Uncertain
- 1.0 = Strong evidence of AI-generated writing

**Limitations:**

- May incorrectly classify polished human writing as AI.
- Can be fooled by AI text that has been heavily edited by a person.

---

### Signal 2 – Stylometric Heuristics

**Purpose:** Measures structural writing characteristics that often differ between AI and human writing.

Metrics include:

- Sentence length variance
- Type-token ratio (vocabulary diversity)
- Punctuation density

**Output:** A normalized score from **0.0 to 1.0**, where higher values indicate writing that appears more AI-generated.

**Limitations:**

- Formal academic writing may resemble AI writing.
- Short submissions provide too little data for reliable analysis.
- Poems and lyrics often break normal writing patterns.

---

### Combining the Signals

The final confidence score is calculated using weighted averaging:

Confidence Score =

```
(Groq Score * 0.6) + (Stylometric Score * 0.4)
```

The Groq score is weighted more heavily because it captures semantic meaning, while stylometric heuristics only analyze writing structure.

---

## Uncertainty Representation

The system avoids making a binary AI-or-human decision whenever possible.

Confidence scores are interpreted as:

| Score Range | Classification       |
| ----------- | -------------------- |
| 0.80 – 1.00 | Likely AI-generated  |
| 0.40 – 0.79 | Uncertain            |
| 0.00 – 0.39 | Likely Human-written |

### Meaning of a 0.60 Score

A confidence score of **0.60** means that the detection signals disagree or provide mixed evidence. Rather than labeling the content as AI-generated, the system reports it as **Uncertain**.

This approach reduces false positives because incorrectly labeling human writing as AI-generated is considered more harmful than failing to detect AI-generated content.

## Transparency Label Design

### High-Confidence AI

> **Likely AI-Generated**
>
> Our analysis found strong evidence that this content was generated with AI.
>
> **Confidence:** High

---

### High-Confidence Human

> **Likely Human-Written**
>
> Our analysis found strong evidence that this content was written by a human.
>
> **Confidence:** High

---

### Uncertain

> **Uncertain Attribution**
>
> Our analysis found mixed signals and cannot confidently determine whether this content was written by a human or generated by AI.

## Appeals Workflow

### Who Can Submit an Appeal?

Only the creator who originally submitted the content.

### Required Information

- Content ID
- Creator reasoning explaining why they disagree with the classification

### System Actions

When an appeal is submitted, the system will:

1. Verify the content exists.
2. Save the creator's explanation.
3. Change the content status to **under_review**.
4. Add an appeal entry to the audit log.
5. Return a confirmation response.

### Human Reviewer View

A reviewer will see:

- Content ID
- Creator ID
- Original classification
- Confidence score
- Individual detection signal scores
- Creator's appeal explanation
- Current review status

---

## Anticipated Edge Cases

### Edge Case 1 – Poetry

Poems often contain repeated words, unusual formatting, and simple vocabulary. The stylometric heuristics may incorrectly classify this writing as AI-generated.

### Edge Case 2 – Academic Essays

Research papers and formal essays often have consistent sentence structure and vocabulary, making them appear similar to AI-generated writing.

### Edge Case 3 – Very Short Text

Messages such as "Great job!" or "Thanks!" contain too little information for reliable analysis.

### Edge Case 4 – Edited AI Content

If AI-generated text has been significantly rewritten by a human, the Groq model and stylometric heuristics may disagree, producing an uncertain result.

# Architecture

```
                POST /submit
                      |
               Receive Submission
                      |
        +-------------+-------------+
        |                           |
        |                           |
   Groq LLM Signal          Stylometric Signal
        |                           |
        +-------------+-------------+
                      |
             Confidence Calculator
                      |
            Transparency Label
                      |
                Write Audit Log
                      |
               Return JSON Response


                POST /appeal
                      |
              Validate Content ID
                      |
            Save Creator Reasoning
                      |
          Update Status: under_review
                      |
              Write Audit Log
                      |
             Return Confirmation
```

### Architecture Narrative

A submission enters the system through the **POST /submit** endpoint. The submitted text is analyzed by both the Groq LLM classifier and the stylometric heuristic analyzer. Their scores are combined into a single confidence score, which determines the transparency label shown to the user. The final result is stored in the audit log before being returned to the client.

If the creator disagrees with the classification, they submit an appeal through **POST /appeal**. The system records the creator's explanation, updates the submission status to **under_review**, logs the appeal, and returns a confirmation response.
